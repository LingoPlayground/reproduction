新方案：基于 ASR 时间轴的视频重生成 Pipeline

核心思路：原剧完整视频是基础，ASR 时间戳决定剪辑位置，画布节点仅用于视觉参考

📊 已有数据资产（Stage 1 输出，当前未被 Stage 3 利用）
ScriptShot（多模态 LLM 分析视频帧 + ASR 得出）：
shot_number, start_seconds, end_seconds, scene_description, shot_type, camera_movement

ScriptLine（ASR 精确时间戳填充）：
line_id, speaker, dialogue, start_seconds, end_seconds, asr_utterance_index, asr_word_start_seconds, asr_word_end_seconds

新 Pipeline 流程

Stage 1（不变）: 剧本提取
原剧视频 → Azure ASR + 多模态 LLM → ScriptOutput（含 shot 边界 + line 时间戳）
✅ 完全复用，无需改动
Stage 1b（新增）: 场景检测增强
原剧视频 → PySceneDetect → 算法级镜头边界
与 Stage 1 的 LLM shot 边界交叉验证 → 更精确的 cut point
可选步骤，对空镜/无声过渡场景更准确
Stage 2（不变）: 剧本改写
ScriptOutput → FullRewriter (CEFR 分级) → 改写 JSON（保留 start_seconds/end_seconds）
✅ 完全复用，无需改动
Stage 3（重写）: 剪辑计划生成
不再做台词→节点文本匹配。改为：
① 读取 ScriptOutput + 改写 JSON → 确定每个 shot 的时间范围
② 对有改写的 shot：找到对应画布节点的参考图 + prompt → seedance 生成新视频
③ 对无改写的 shot：标记为「复用原剧片段」
④ 输出 Timeline 剪辑计划：{shot_id, start, end, source: "original"|"seedance", video_url, reference_images, prompt}
✅ 画布节点匹配仅用于参考图获取，不影响剪辑结果
Stage 4（重写）: 视频组装
① 从原剧视频按 Timeline 截取："复用原剧" 片段
② seedance 生成 "改写" shot 的新视频
③ ffmpeg concat 按时间顺序拼接所有片段 → 最终视频
✅ 拼接点的准确度由 ASR 时间戳保证，不依赖画布节点

剪刀位置确定策略

主信号：Stage 1 的 ScriptShot 边界
多模态 LLM 已经通过观看视频 + 阅读 ASR 确定了 shot 的 start/end。这是逻辑镜头边界， 而非纯画面切割 —— LLM 理解叙事结构。

辅助信号：PySceneDetect
纯画面 Content-Aware 检测，对 LLM 可能遗漏的：
  • 无台词的过渡空镜
  • 快速切换的蒙太奇镜头
  • LLM 因视频压缩（480p, 8fps）导致的边界模糊
融合方式：LLM 边界 ± 容差窗口内（如 0.5s），若 PySceneDetect 也检测到切换 → 高置信度。 若仅 PySceneDetect 检测到但 LLM 未标注 → 检查是否是过渡镜头/空镜。

画布节点的新定位

旧定位（问题）

  • 视频源 — 必须精确匹配台词到节点
  • 废片内容导致匹配不可靠
  • 匹配失败 = 视频拼接错误

新定位（改进）

  • 视觉资产库 — 提供参考图 + prompt 模板
  • 匹配到「够用」即可，不需要逐行精确
  • 匹配失败 → 用原剧片段，不影响拼接
