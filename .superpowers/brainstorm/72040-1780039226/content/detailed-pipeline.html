新方案：完整 Pipeline 设计

核心原则：原剧视频 = 时间轴基准 | 画布节点 = 视觉资产库 | ASR 时间戳 = 剪刀位置

🎯 画布节点的新角色：Prompt 片段提取器

匹配方式
用 台词文本 + 场景描述（非逐词匹配，语义匹配）找到对应画布节点。
一个画布节点的 prompt 可能覆盖多个 shot（如节点 0e19c709 覆盖 shot 5 + 6）。

关键操作：Prompt 片段提取
不是把多个 shot 的改写台词塞进原 prompt，而是：
① 在节点长 prompt 中定位目标 shot 对应的片段
② 提取该片段的视觉描述（镜头、景别、运镜、表情、动作）
③ 替换其中的原台词为改写台词
④ seedance 用这段「裁切后的 prompt + 节点参考图」生成新视频

✅ Prompt 截取（你的方案）

  • 每个 shot 独立生成，鲁棒性强
  • 废片节点的 prompt 片段也能产出合理视频
  • 不需要精确逐行匹配，语义匹配够用

❌ 多台词合并（不采用）

  • 废片节点 = 错误台词/画面 → 输出错误
  • 多 shot 的改写台词塞入一个 prompt → LLM 理解偏差

📐 PySceneDetect 的三个角色

角色 1：剪辑点精修（主用途）
Stage 1 的 LLM 给出了 ScriptShot 的逻辑边界（start/end 秒）。
PySceneDetect 在原剧视频上给出画面级镜头切换点（帧精度）。
融合策略：在 LLM 边界 ±N 秒窗口内，选择最近的 PySceneDetect 切换点作为最终 cut。
这样既能保证语义正确（LLM 理解叙事），又能保证画面干净（算法检测切换）。
角色 2：画布节点内部分段
一个画布节点的 prompt 覆盖多个 shot 时，PySceneDetect 分析该节点的生成视频， 检测内部镜头切换点 → 辅助判断 prompt 中「镜头1」「镜头2」等不同段落 各自对应哪个 ScriptShot → 帮助 prompt 片段提取更精准。
角色 3：关键帧提取
对于没有画布节点的 shot（或匹配失败的 shot），从原剧视频中按 PySceneDetect 检测到的镜头边界提取关键帧，作为 seedance 的参考图 fallback。
